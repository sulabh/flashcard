Development & Environment Setup | 開発環境・セットアップマニュアル

01. Prerequisites | 開発環境

日本語
  • Flutter SDK: 3.10.x 以上 推奨。
  • Dart SDK: 3.x 以上。
  • IDE: VS Code または Android Studio (Flutterプラグイン必須)。
  • Flavor設定: Android/iOSともに `free` と `paid` の2つのFlavorが構築済みです。
English
  • Flutter SDK: 3.10.x+ recommended.
  • Dart SDK: 3.x+.
  • IDE: VS Code or Android Studio with Flutter plugin.
  • Flavors: Bi-flavor architecture (`free`, `paid`) is fully configured for Android and iOS.

02. Essential Commands | 主要コマンド

日本語

デバッグ実行

Free版（広告あり）の実行:

flutter run --flavor free -t lib/main_free.dart

Paid版（広告なし）の実行:

flutter run --flavor paid -t lib/main_paid.dart

ビルド

flutter build apk --flavor free -t lib/main_free.dart
flutter build web --no-wasm-dry-run
English

Debug Run

Execute Free Version (with Ads):

flutter run --flavor free -t lib/main_free.dart

Execute Paid Version (No Ads):

flutter run --flavor paid -t lib/main_paid.dart

Production Build

flutter build apk --flavor free -t lib/main_free.dart
flutter build web --no-wasm-dry-run

03. Architecture | アーキテクチャ

日本語
  • State Management: hooks_riverpod を使用。
  • Database: sqflite (SQLite) によるローカル永続化。
  • Localization: l10n.yaml と .arb ファイルによる多言語管理。
  • Routing: go_router による宣言的ルーティング。
English
  • State Management: Powered by hooks_riverpod.
  • Database: Local persistence via sqflite (SQLite).
  • Localization: Managed via l10n.yaml and .arb template files.
  • Routing: Declarative routing using go_router.

04. Data Schema | データベース構造

日本語

flashcards テーブルの主なカラム:

  • id: ユニークID。
  • frontHtml: カード表面（Furigana/Fraction対応）。
  • backHtml: カード裏面。
  • category: 大分類（科目）。
  • ageGroup: 中分類（年齢）。
  • unit: 小分類（第1期/第2期など）。
English

Key Columns in flashcards table:

  • id: Unique identifier.
  • frontHtml: Card front content (tags allowed).
  • backHtml: Card back content.
  • category: Subject level.
  • ageGroup: Middle filter level.
  • unit: Specific units/chapters.

05. Deployment | ストア配信ガイド

日本語

Androidアプリの署名

1. keytool を使用してリリース用のKeystoreを生成します。

2. android/key.properties ファイルを作成し、パスワード等を記述します。

3. 以下のコマンドでApp Bundleを生成し、Google Playにアップロードします。

flutter build appbundle --flavor free -t lib/main_free.dart

iOSアプリの配信

1. Xcodeで Runner.xcworkspace を開きます。

2. 各Flavorの Signing & Capabilities を設定します。

3. Product > Archive を実行し、App Store Connectへアップロードします。

English

Android Release

1. Generate a release Keystore using keytool.

2. Create android/key.properties with your key details.

3. Generate the Android App Bundle (AAB):

flutter build appbundle --flavor free -t lib/main_free.dart

iOS Release

1. Open Runner.xcworkspace in Xcode.

2. Configure Signing & Capabilities for the desired flavor.

3. Perform Product > Archive and upload to App Store Connect.
